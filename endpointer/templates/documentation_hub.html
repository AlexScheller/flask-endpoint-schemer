Notes regarding documentation

Some routes will document certain response codes as return a JSON response, even though they do not. For example, /user/auth/token lists 401 as returning { error: "Unauthorized" }, when in reality it returns a simple, unformatted unauthorized. This is because 401, as well as other responses are overloaded, and the documentation generator makes use of certain default values for these common responses.

Rest assured that any response warranting (of course this is the opinion of the developer) further inspection beyond the status code will have accurate example JSON.
